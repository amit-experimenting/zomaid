My recommendation: a (1-per-screen wizard). Matches the mockup you shared. The questions are short and the answers fit on one screen each — exactly the case where wizard pattern shines (low cognitive load, clear progress, easy to back out). (b) is faster but loses the "this is meaningful, please think" framing — and on a mobile keyboard, scrolling 8 questions feels long. (c) is technically tidiest but the questionnaire becomes invisible filter UI rather than a meaningful onboarding step. Since this is a first-time-only commitment, (a)'s extra screens are worth the polish.
